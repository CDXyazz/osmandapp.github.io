Custom Package or your own plugin for OsmAnd

May 15, 2020

OsmAnd is "Swiss knife" among similar application. We always try to create very useful and flexible application.

Old OsmAnd users are our gold team, because it is not easy to start using OsmAnd for 100 percent at once.

Earlier we added customization of your navigation profiles. At last version of OsmAnd (3.7) you can customize Configure map menu, Drawer... At general all time you can send your navigation profile to your friends with all your settings.

But if you want to send more settings, not only profile, but rendering file, fonts, routings, maps... Now we added this opportiunity. You can create your own plugin file for this. Plugin file is special zip file whick has .osf format. At this article we give your tutorial how to do your own plugin for OsmAnd.

Let's start!

OSF-file is a zip-file which contains json-files and folders. When you click to OSF-file OsmAnd

Read more about JSON-format here. You can open and edit this format any txt editors. We will use Sublime Text editor in our example.

We want to add that we made the opportunity to modify JSON file for all our users not only programmist.

Our general JSON file is "items" contains commands for import JSON-files, obf-files, icons...

At the first part of items.json we can create and add your plugin. The first type will be "PLUGIN".

Note: you must create and add for each type this line: "pluginId":"test.plugin". Where test.plugin is a name of your package.

"type" is place where added files.

For import icon image to of your plugin to OsmAnd you can create folder (in our example "res" folder) where we have our image files. This type names "RESOURSES".
